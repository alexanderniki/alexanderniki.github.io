# The Product Handbook

The Product Handbook - это справочник по созданию цифровых продуктов.

Использование этого справочника предполагает, что бизнес-проблема уже сформулирована и бизнес-требования к разрабатываемому новому продукту уже существуют.

По каждой из тем, затронутых в справочнике, написано множество книг и статей.
Цель этого проекта - дать краткую информацию, похожую на словарь терминов. Обозначить используемые при проектировании категории, поятия и термины и показать взаимосвязи между ними (т.н. онтологию).
Так, чтобы у вас была "отправная точка" для дальнейшего изучения.
 

## Создание digital-продуктов

### Философия

- [Кайдзен](product-phylosophy-kaizen.md) - Философия или практика, которая фокусируется на непрерывном совершенствовании процессов производства
- [Философия Python](product-phylosophy-python.md) - Набор основополагающих принципов языка программирования [Python](https://python.org)
- [Философия UNIX](product-phylosophy-unix.md) - Основополагающие принципы, заложенные при проектировании ОС Unix. Т.н. Unix way.
- [Zenware](product-phylosophy-zenware.md) - Подход к разработке приложений, при котором особое внимание уделяется тому, чтобы помочь человеку оставаться максимально сфокусированным на своей задаче
- [YAGNI - You Aren't Gonna Need It](product-phylosophy-yagni.md) -  Принцип проектирования ПО, при котором в качестве основной цели и/или ценности декларируется отказ от избыточной функциональности
- [Чем хуже, тем лучше](product-phylosophy-worse-is-better.md) - Подход к разработке ПО, объявляющий простоту реализации и простоту интерфейса более важными, чем любые другие свойства системы.
- [Дизайн-подход MIT](product-phylosophy-mit-design-approach.md) - Подход к разработке ПО, обычно противопоставляемый подходу "Чем хуже, тем лучше"
- [DRY - Don't Repeat Yourself](product-phylosophy-dry.md)
- [UNPHAT](product-phylosophy-unphat.md)
